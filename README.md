## **面向大模型智能体的记忆系统**

本项目基于开源大模型记忆管理工具mem0开发，对企业智能体应用场景做了特化开发，意在满足企业需求。

### 项目背景

随着大语言模型在问答、写作、数据分析、流程规划和工具调用等场景中的应用不断深 入，智能体已不再只是完成单次问答，而是逐步承担起任务跟踪、过程协同、信息整理和辅 助决策等连续性工作。

在这类任务中，智能体需要长期保存用户偏好、任务背景、阶段进展、 历史决策和反馈意见，并在后续对话或跨会话任务中准确调用这些信息。

但现有智能体通常 主要依赖当前上下文窗口或简单会话缓存，一旦对话轮次较长、任务周期拉长，或者用户重 新开启会话，就容易出现历史信息遗忘、关键事实丢失、任务状态断裂、重复询问和前后回 答不一致等问题。 

### 项目概述

本项目面向大模型智能体在长周期交互和连续任务执行过程中存在的上述记忆能力不足问题，建设一套独立、通用、可接口化调用的智能体记忆管理系统，为各类智能体提供记 忆写入、记忆生成、记忆管理、记忆检索和上下文返回能力。系统能够接收智能体产生的用 户输入、智能体回复、历史会话、任务过程、执行结果及相关元数据，并将这些原始交互数 据转化为可存储、可检索、可复用的标准化记忆内容，使智能体具备更稳定的长期记忆保持 和任务延续能力。

系统整体围绕“记什么、怎么存、如何找、怎样用”四个核心问题展开建设。 （1）在记忆生成方面，系统不是简单保存完整聊天记录，而是从多轮交互中提取真正 有持续价值的信息，包括用户偏好、关键事实、任务目标、阶段进展、历史决策、约束条件、 待办事项和修改意见等，并对重复、相似或低价值内容进行过滤和融合，减少记忆库冗余。

（2）在记忆管理方面，系统将不同来源的信息统一建模为记忆单元，并按照用户级、 会话级、任务级和智能体状态级等不同层次进行组织，保证记忆既能服务当前对话，也能支 撑跨会话、跨任务的长期使用。

（3）在记忆检索方面，系统综合语义相似度、关键词匹配、时间信息、记忆类型、用 户标识、场景标识和任务标识等多种信号，从历史记忆中召回与当前问题最相关的内容，并 对结果进行排序和筛选，避免无关记忆干扰智能体回答。

（4）在记忆召回方面，系统可以将检索到的记忆整理为结构化数据，也可以生成可直 接注入大模型提示词的上下文片段，使上层智能体能够方便地调用历史信息。 

上述问题可通过调用现有开源工具 Mem0 实现，本实训的主要工作是完成系统功能的 前后端开发，以及 Mem0 的调用，实现上述效果，演示 demo。
